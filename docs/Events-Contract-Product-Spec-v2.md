# Events Contract（Now）产品规格文档

> **模块定位**：独立功能板块，与现有 Instant Market 完全隔离  
> **当前阶段**：前端产品外形 + Mock 数据演示  
> **可参考 Demo**：https://errance.github.io/TurboFlow/events

---

## 一、产品概述

Events Contract（Now）是一个短周期事件合约模块。用户对加密资产的短期价格走势进行预测（涨/跌），在极短的回合周期内完成下注 → 持仓 → 结算的完整流程。

体验定位：**回合制沉浸游戏**，不是传统交易工具。强调节奏感、紧迫感和揭晓仪式感。

---

## 二、用户旅程

用户从进入 Events 页面到完成一个完整回合的路径：

```
进入页面
  → 浏览行情（价格、走势、全局动态）
  → 决策下单（选时长 → 输金额 → 押方向）    ← 目标 3–5 秒完成
  → 等待结算（持仓卡片实时展示倒计时和预估盈亏）
  → 临战阶段（最后几秒，紧张感拉满）
  → 结算揭晓（全屏动画展示结果）
  → [可选] 仪式庆祝（连胜/逆转时触发额外庆祝）
  → 回到页面，开始下一回合
```

用户可以同时持有多笔不同时长的单子，每笔独立倒计时、独立结算。

## 三、导航与入口

- 主导航栏新增 **"Events"** Tab，带新功能角标
- Portfolio 页面内新增 **"Events"** Tab，展示用户的 Events 相关汇总和历史

## 四、页面布局

### 4.1 区域划分

页面由以下区域组成：

| 区域 | 职责 |
|------|------|
| **排行榜** | 可折叠区域，展开后展示完整排行榜 |
| **行情区** | 价格信息 + 微走势图 |
| **下单区** | 选择时长、金额、方向，一键下注 |
| **持仓区** | 当前活跃的持仓卡片 + 全局玩家动态 |

### 4.2 布局原则

- **桌面端**：行情区和下单区应能同时可见。下单区在滚动时保持可见
- **移动端**：各区域纵向堆叠
- **行情区的微走势图与下单区顶部对齐**（桌面端）

---

## 五、行情区

### 5.1 价格信息

- 资产选择器：下拉切换不同交易对（当前支持 BTC/USDT、ETH/USDT），切换不影响已有持仓
- 实时价格：大号展示，价格跳动时有视觉反馈
- 趋势指示：简要标明当前价格动量方向

### 5.2 微走势图

用精简折线图取代传统 K 线（对短周期回合而言 K 线信息冗余）：

- 只展示价格走势轮廓，不需要坐标轴和网格
- 时间窗口与用户选择的下注时长对齐

### 5.3 微走势上的叠加信息

**用户下注标记：**
- 用户每次下单后，入场点位在走势图上标记出来
- 进行中的单子和已结算的单子应有视觉区分
- 已结算的单子标记上展示结算结果（赢/输和盈亏金额）

**全局结算浮标：**
- 其他玩家的结算结果以飘字形式在走势图上短暂出现
- 展示玩家名和盈亏金额
- 盈利和亏损的飘字应有方向感上的区分（如盈利向上飘、亏损向下飘）
- 自然散射，避免呆板排列，短暂停留后消失
- **作用**：营造"交易大厅"的热闹氛围

---

## 六、下单区

### 6.1 信息与操作

从上到下包含：

1. **可用余额**
2. **时间档位选择**：1min / 5min / 15min
3. **金额输入**：输入框 + 快捷金额按钮 + 明确展示的金额范围限制
4. **收益预览**：分别展示 Higher 和 Lower 方向的预期收益。两个方向的信息在视觉上与对应方向的颜色一致
5. **下注按钮**：Higher / Lower 两个按钮，**点击即确认下注**，不提供撤销

### 6.2 下单反馈

- 下单成功：轻量 Toast 提示，标明资产、方向、金额、时长
- 下单后金额清空，准备下一单
- 余额不足或金额不合规时按钮不可用

---

## 七、持仓区

### 7.1 Tab 结构

| Tab | 内容 |
|-----|------|
| **My Positions** | 当前用户的活跃持仓卡片列表 |
| **All** | 全局玩家的实时下注和结算动态流 |

**History 按钮**始终可见（不随 Tab 切换隐藏），点击打开历史记录面板。

### 7.2 My Positions

- 以卡片形式展示每笔活跃持仓
- 按剩余时间排序，快到期的排在前面
- 即将到期的单子自动冒泡到列表顶部（平滑过渡）

### 7.3 All（全局动态）

- 实时展示其他玩家的下单和结算记录
- 新记录有入场动画
- **作用**：让用户感受到平台上有大量玩家在同时参与

---

## 八、持仓卡片

每张卡片是一个独立的"迷你回合舞台"。

### 8.1 卡片信息

- 进度指示器：展示剩余时间
- 方向标签：Higher / Lower
- **预估 PnL**（核心信息，突出展示）：实时跟随价格变化翻转，标注"如果此刻结算"的含义
- 辅助信息：入场价 / 当前价 / 下注金额 / 回报率

### 8.2 倒计时体验

这是产品的核心体验差异点——借鉴游戏中抽卡、开箱、孵蛋的递进式紧张感：

| 阶段 | 体验 |
|------|------|
| **常规** | 进度平稳消耗，卡片正常展示 |
| **临战**（最后几秒） | 卡片整体进入紧张状态——**通过进度指示器、倒计时数字、卡片自身的视觉变化，让用户强烈感受到"即将揭晓"的紧迫感和刺激感**。不使用任何额外的徽章或贴片，紧迫感完全由时间容器和卡片自身效果来表达 |
| **结算** | 进入结算揭晓流程 |

---

## 九、结算揭晓

### 9.1 核心理念

结算揭晓是**打断式沉浸体验**——当订单结算时，默认用全屏动画展示结果，不论用户当前在做什么。这是回合制体验的核心设计，不是打扰。

### 9.2 揭晓内容

以全屏遮罩展示，背景保留底层页面若隐若现：

- **结果标题**：WIN 或 LOSS
- **盈亏金额**（核心信息，最突出展示）
- **交易摘要**：资产、方向、入场价到结算价的变化、下注金额和时长

### 9.3 胜负视觉差异

- **WIN**：庆祝感，可以有光效、粒子等强化表现
- **LOSS**：克制，不要过于打击用户

### 9.4 队列与退出

- 点击任意区域或键盘可关闭当前揭晓
- 多单同时到期时按时间顺序**逐个揭晓，不合并**
- 跳过只跳一个，不能一次跳过全部
- 展示一段时间后自动关闭

### 9.5 效果等级

用户可选择三档：

| 等级 | 行为 |
|------|------|
| **On** | 完整的全屏揭晓动画（默认） |
| **Minimal** | 不全屏，仅在卡片内部做简短反馈 |
| **Off** | 静默更新，无动画 |

---

## 十、仪式系统

在特定条件下，结算揭晓之后叠加额外的庆祝效果：

| 触发 | 条件 |
|------|------|
| **连胜** | 连续赢达到一定次数 |
| **逆转** | 连续输多次后首次赢 |

仪式的表现应该比普通结算揭晓更强烈、更持久，给用户"高光时刻"的记忆点。仪式**不产生任何持久性的 UI 元素**——只在揭晓瞬间展示。

用户可通过 Hype Mode 开关关闭仪式。

---

## 十一、Effects 设置

提供两个用户可控的开关：

| 设置 | 作用 |
|------|------|
| **Settlement Reveal** | 控制结算揭晓的强度（On / Minimal / Off） |
| **Hype Mode** | 控制连胜/逆转仪式（On / Off） |

入口应在 Events 页面上明确可见。

---

## 十二、排行榜

### 12.1 展示形式

页面内可折叠区域。收起时展示简要信息，展开后全宽展示完整榜单。

### 12.2 三个独立榜单

通过 Tab 切换三个独立排名：

| 榜单 | 排名依据 |
|------|---------|
| **Total Profit** | 总盈利金额 |
| **Best Trade** | 单笔最高盈利 |
| **Win Streak** | 最高连胜记录 |

每个榜单各自排名，展示主指标并附带其他指标作为辅助参考。支持分页浏览。

---

## 十三、历史记录与珠盘路

### 13.1 History 面板

点击 History 按钮打开，包含：

1. **汇总统计**：总盈亏、胜率、胜负场次
2. **珠盘路**
3. **逐笔记录列表**

### 13.2 珠盘路（Bead Road）

借鉴百家乐珠盘路的标准网格可视化：

- 网格式棋盘布局，每格一个圆点代表一局结果
- 从上到下、从左到右按时间顺序填入
- 赢和输用不同颜色区分
- 超出宽度时可横向滚动，自动聚焦到最新数据

**作用**：让用户一眼识别胜负趋势和连续性，是赌场类产品的标准配置。

---

## 十四、全局玩家数据

系统在后台持续生成模拟的全局玩家活动，数据呈现在以下位置：

| 呈现位置 | 形式 |
|---------|------|
| **All Tab** | 实时交易动态流 |
| **微走势浮标** | 结算飘字 |
| **排行榜** | 三榜排名 |

**目的**：营造热闹的交易氛围，让用户感知到全局活跃度，制造社交临场感。

---

## 十五、异常与边界

| 场景 | 处理 |
|------|------|
| 价格未更新 | 预估盈亏展示加载/等待状态 |
| 多单同时到期 | 逐个揭晓，不合并 |
| 余额不足 | 下注按钮不可用 |

---

## 十六、Demo 说明

当前 Demo 采用以下简化：

- 所有时长档位实际倒计时统一缩短为演示用时长
- 价格数据为随机模拟
- 全局玩家数据为系统自动生成
- 所有交易在本地完成，不涉及后端
