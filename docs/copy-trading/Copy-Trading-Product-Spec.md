# 合约跟单产品规格文档

> **模块定位**：合约交易下的跟单（Copy Trading）能力，跟单账户与合约账户独立统计  
> **适用范围**：仅支持 **USDT 计价合约**交易对  
> **文档范围**：本文档描述业务逻辑与规则，不涉及具体 UI 布局或控件文案；页面与交互以设计稿为准  
> **文档版本**：v2.2（2026-03-09）  
> **信息来源**：PRD《合约跟单详解》、交易员/跟单员业务流程图、产品访谈、Slack 决策记录

---

## 一、产品概述

合约跟单允许**跟单者**选择**带单员**，按带单员的开仓订单自动跟单开平仓；带单员可从跟单者的盈利中获得**分润**。

**角色定义**

- **带单员**：申请通过后可在交易页通过专用带单路由发起带单；带单员开仓时，系统自动为其跟随者按各自配置开仓；带单员平仓时，关联跟单者仓位一并平仓。
- **跟单者**：在带单广场发现并选择带单员，完成跟单设置后由系统自动跟单开平仓；可查看当前跟单持仓、历史记录，并管理资金。

**账户体系**

- 跟单账户与合约账户**相互独立**，两边资金流水单独统计。
- 跟单者开始跟单时，资金从合约账户转入跟单账户；停止跟单后，剩余资金转回合约账户。
- 所有跟单仓位均以 **USDT** 计价结算，分润计算全程使用 USDT，精度与平台标准一致。

**核心价值**

跟单者省心复制带单员策略；带单员通过分润获得额外收益。

## 二、带单员流程与规则

### 2.1 申请成为带单员

**系统前置校验**（任一不满足则申请失败）：

1. 账户下所有合约**无仓位、委托单、计划单**
2. 账户**未跟随他人订单**

两项校验均通过后进入**人工审核**：

- 审核通过 → 申请成功，账号获得带单员资格
- 审核拒绝 → 申请失败，需重新申请

> 具体审核标准（如收益率、交易笔数、盈利笔数等）由运营人工把控。

### 2.2 带单策略配置

每个带单员拥有**一套**带单策略，可配置以下参数：

| 配置项 | 说明 |
|--------|------|
| 最低跟单金额 | 带单员自行设置，跟单者资产不满足则无法跟随 |
| 分润比例 | 按带单员等级决定（见第四章），待运营配置 |
| 锁定期 | 可选开关。开启后：跟单者在锁定期内不可停止跟单、不可转入或转出资金。关闭后按正常资金规则执行 |
| 公域/私域 | 公域：列表可见且信息公开；私域：列表可见但详情隐藏，跟单者需输入邀请码解锁 |

**展示规则**

- 公域带单员正常显示在带单广场，支持搜索；
- 私域带单员在广场列表中可见，但具体统计数据、交易记录等信息不可见，需邀请码解锁；
- 带单员账户的**合约总权益 ≥ 500 USDT** 时，系统考虑将其展示在列表；是否展示及排名由人工审核决定。

**私域邀请码管理**

- 带单员可自行生成邀请码，可生成**多个不同的码**，每个码可供**多人使用**
- 带单员可随时 **disable（禁用）** 已生成的码；禁用后该码不可再被新跟随者使用
- 已通过该码加入的跟随者**不受影响**，继续保留跟随关系

### 2.3 发起带单

带单员在交易页通过**专用带单路由**开仓，该路由下的订单即为带单仓位，系统自动触发跟单逻辑。通过普通交易路由开仓的订单为带单员自有仓位，**不触发跟单**。

- 同一合约同一方向可创建多笔带单订单。
- 带单员可在发起带单时**设置止盈止损**，展示止盈止损价供跟单者参考。

**带单仓位的平仓方式**

带单员平仓后，当前仓位关联的所有跟单者仓位一并平仓。支持以下方式：

- 市价平仓
- 市价全平
- 止盈止损触发平仓

平仓同时生成**明细订单**与**汇总订单**。

### 2.4 移除跟随者

带单员可主动移除跟随者：

- 被移除后，跟单者**不再跟随新的带单订单**
- 被移除跟单者的**已有跟单仓位仍随带单员自动平仓**（或跟单者手动平仓）
- 带单员主动移除跟随者属于主动行为，被移除后的仓位产生的盈利**不计入分润**，带单员放弃相应分润权益

### 2.5 带单员退出

**停止带单（保留账号）**
- 停止接收新跟单
- 系统立即将所有跟单者的仓位**市价全平**
- 分润按正常结算流程处理（正常结算，不作废）

**注销带单员身份**
- 停止带单，处理同上
- 账号退出带单员列表

> 带单员退出不影响已发生的交易事实，分润照常次日结算。

### 2.6 平台封禁带单员

紧急处理流程（与主动退出不同）：

1. 立即**冻结**该带单员的所有带单功能，不再触发新跟单
2. 所有关联跟单者仓位**立即市价强制全平**
3. **冻结**所有待结算分润，等待调查结果决定是否发放
4. 通过**站内信**通知所有跟单者封禁原因及后续补偿措施

## 三、跟单者流程与规则

### 3.1 发现带单员

带单广场提供三个 Tab：

| Tab | 说明 |
|-----|------|
| **公域** | 所有通过审核的公开带单员，支持搜索与筛选 |
| **私域** | 私域带单员的列表入口，具体信息隐藏，需输入带单员提供的邀请码解锁后方可查看详情并跟单 |
| **收藏** | 跟单者收藏的带单员列表，纯展示，无特殊业务逻辑 |

### 3.2 开始跟单前的校验

系统依次校验（任一失败则无法跟随）：

1. 跟单者资产 ≥ 带单员设置的**最低跟单金额**
2. 带单员**近 30 日有开仓记录**（无记录时提示并阻止跟随）

### 3.3 跟单设置参数

跟单者在确认跟随前需完成以下设置：

**① 跟单类型（必选）**

| 类型 | 说明 |
|------|------|
| **Fixed Amount（固定金额）** | 每笔跟单订单使用固定 USDT 金额，适合控制单笔风险；转入额外资金不影响每笔跟单金额 |
| **Fixed Ratio（固定比例）** | 设置总投资金额，系统按比例自动跟随带单员每笔仓位大小；转入额外资金后，每笔跟单金额按新余额等比例增大 |

总投资金额范围：10 – 300,000 USDT（参考币安，具体上下限以运营配置为准）。

**② 现有持仓处理（必选）**

| 选项 | 说明 |
|------|------|
| 不复制现有持仓 | 仅跟随新开仓，带单员已有仓位不复制 |
| 复制更好入场 | 当前市价优于带单员入场价时才复制现有仓位 |
| 复制全部现有持仓 | 以市价复制带单员所有已有仓位 |

**③ 交易对白名单（可选）**

- 自定义跟随哪些交易对，未选中的交易对不跟单
- 可选择「自动加入新上市交易对」

**④ 保证金模式**

- 固定模式：全仓（Cross）或逐仓（Isolated）
- 跟随带单员：与带单员保持一致

**⑤ 杠杆倍数**

- 固定倍数：1x – 10x
- 跟随带单员：与带单员保持一致

**⑥ 最大入场滑点**

- 范围：0.1% – 3%
- 默认：BTC/ETH 为 0.5%，其他交易对为 1.5%
- 仅保护入场价，不保护平仓价

**⑦ 仓位级止盈止损（TP/SL）**

可选，开仓后系统自动为每个仓位挂 TP/SL 单。ROI% 为正数，方向由系统根据多空自动判断：

| 方向 | TP 触发价 | SL 触发价 |
|------|-----------|-----------|
| 多头（Long） | 入场价 × (1 + ROI% / 杠杆) | 入场价 × (1 − ROI% / 杠杆) |
| 空头（Short） | 入场价 × (1 − ROI% / 杠杆) | 入场价 × (1 + ROI% / 杠杆) |

> 注意：止损比例过小时，触发价可能与当前市价倒挂导致挂单失败。编辑跟单设置后的 TP/SL 仅对新仓位生效，已有仓位的 TP/SL 不会被自动更新或取消。

**⑧ 总止损（账户级）**

当跟单账户的保证金余额亏损达到设定值时，系统自动市价全平所有跟单仓位并停止跟单。支持两种设置方式：

**按 USDT 金额：**
- 设定一个止损触发余额绝对值
- 例：投入 100 USDT，设止损余额 40 USDT → 余额跌至 40 USDT 时触发

**按百分比：**
- 公式：止损触发值 = 当前余额 × (1 − 止损率%)
- 例：余额 100 USDT，设止损率 60%（即最多可接受亏损 60%）→ 触发线 = 100 × (1−60%) = 40 USDT
- 当余额跌至 40 USDT 时触发

**资金操作后的止损重算（与跟单类型无关）：**

每次转入资金后，百分比模式的止损触发值按**当前余额**重新计算：

- 新止损触发值 = 转入后新余额 × (1 − 止损率%)
- 例：余额 100 USDT，设 60% 止损率（触发线 40 USDT）→ 转入 60 USDT 后余额 160 USDT → 触发线重算为 160 × 40% = 64 USDT
- USDT 绝对值模式的止损触发值**不重算**，维持原设定的绝对值

**⑨ 锁定期（如带单员开启了锁定期）**

- 跟单者须确认并接受锁定期条款后方可跟随
- 锁定期内不可停止跟单、不可转入或转出资金

### 3.4 跟单执行逻辑

带单员开仓后，系统对每位跟随者依次执行：

1. **成本检查**：跟单账户资产是否充足？
   - 不足 → 本次跟单失败，**立即触发系统自动取消跟随**
2. **仓位上限分配**：带单金额总计（带单员 + 所有跟单者）是否超过平台该币种可开上限？
   - 未超出 → 各跟单者按实际设置金额开仓
   - 超出 → 以平台上限为准，带单员优先占用，**剩余额度按跟单者金额占比 pro-rata 分配**
   - 分配后若某跟单者的实际可开金额 < 最小开仓量，该笔跟单**跳过**（不触发成本不足的自动取消逻辑）
3. **开仓**：校验通过 → 开仓跟随成功

> **注意**：开仓订单成交多笔时，跟单者**只跟单一次**（按第一笔成交触发）。

带单员强平时，所有跟单者对应的跟单仓位同步触及平仓。

### 3.5 资金管理

| 操作 | 规则 |
|------|------|
| **转入（追加）** | 跟单进行中随时可追加资金；Fixed Ratio 模式下转入后每笔跟单金额等比例增大；百分比止损触发线按新余额重算 |
| **转出** | 跟单进行中**不允许转出**（全仓/逐仓均不可）；如需转出，须先停止跟单，仓位全平后资金自动转回 |
| **锁定期内** | 转入与转出均**不可操作** |

**全仓/逐仓转出说明：**

- **全仓（Cross）模式**：共享保证金，转出会直接降低所有仓位的维持保证金，可能触发强平，跟单期间不允许转出
- **逐仓（Isolated）模式**：各仓位保证金独立，但同样需先停止跟单才可转出

### 3.6 停止跟单

三种取消跟随方式：

| 方式 | 触发 | 已有仓位处理 |
|------|------|-------------|
| **主动取消** | 跟单者手动停止跟随 | **立即市价全平**所有跟单仓位，剩余资金转回 |
| **带单员取消** | 带单员主动移除该跟单者 | 已有仓位继续跟随带单员平仓；跟单者可手动平仓；该部分仓位**不计入分润** |
| **系统取消** | 见下方触发条件 | 已有仓位继续跟随带单员平仓；跟单者可手动平仓 |

**系统自动取消跟随的触发条件：**

1. 跟单者**1 次**因**本金不足**跟单失败
2. 带单员**近 30 日无新开仓**

## 四、结算与分润

### 4.1 结算方式

1. 带单员**平仓**时，若跟随者产生利润且达到分润要求，系统从跟随者账户**预先扣除分润**
2. 于**次日零点**结算，按跟随者当日跟单的**实际总盈利**计算应付给带单员的分润
3. 若实际应付分润 < 已预扣金额，多扣部分**返还**跟随者账户，剩余部分结算给带单员
4. 结算周期：**每日结算**
5. 跟单账户与合约账户独立，资金流水分开统计

### 4.2 分润要求

1. 跟随者与该带单员**已无仓位关联**（当日所有跟随订单均已平仓）时，**次日分润**
2. 若有**未平仓的跟随订单**，则不分润，直至无仓位关联后**次日汇总分润**
3. 分润比例按**带单员等级**决定（待运营配置）；带单员升级后，**升级前的带单按原比例，新开仓的带单按新比例**
4. **分润仅在净盈利为正时产生**；当日所有跟单仓位合并盈亏 ≤ 0 时，分润为 0，不存在带单员倒贴的情况

### 4.3 分润案例

以下案例假设带单员分润比例为 **8%**，所有仓位均为 USDT 计价合约：

**Case 1**：单合约盈利、结算前全部平仓

用户 A 跟随带单员 M 开仓 BTC/USDT，盈利 **200 USDT**。结算点前全部平仓。  
预先扣除：200 × 8% = **16 USDT**  
→ 次日结算时给带单员 **16 USDT**

**Case 2**：多合约有盈有亏、结算前全部平仓

用户 B 开仓 BTC/USDT（盈利 300 USDT）、ETH/USDT（盈利 200 USDT）、SOL/USDT（亏损 150 USDT），结算前全部平仓。  
合并净盈利：300 + 200 − 150 = 350 USDT（净盈利为正，计入分润）  
预扣：350 × 8% = **28 USDT**  
→ 次日结算 **28 USDT** 给带单员

> 若合并净盈利 ≤ 0，则分润为 0。

**Case 3**：盈利单已平、亏损单未平

用户 C 同上三仓，结算点前**盈利单（BTC/USDT、ETH/USDT）已平**，SOL/USDT 亏损单**未平仓**。  
预扣：(300 + 200) × 8% = 40 USDT  
→ 次日**不分润**（有未平仓位，等待最终合并结算）

第 3 日 SOL/USDT 亏损单平仓（亏损 150 USDT）：  
实际净盈利 = 300 + 200 − 150 = 350 USDT  
实际应付分润 = 350 × 8% = 28 USDT  
多扣部分 (40 − 28 = 12 USDT) 返还用户 C  
→ 第 4 日结算 **28 USDT** 给带单员

**Case 4**：盈利单与亏损单均未平仓

→ **不预先扣除，不分润**，等待所有仓位关联结束后再按规则结算

**「预计待分润」字段说明**

- 包括：历史已平仓且盈利订单的待分润部分 + 今日已平仓且盈利的待分润部分
- 平仓收益为正 → 预计待分润增加；平仓收益为负 → 预计待分润减少

### 4.4 带单员退出时的分润

- 停止带单或注销身份触发的市价全平，产生的盈亏**纳入正常分润结算流程**
- 带单员退出原因不影响分润结算，已发生的交易事实不受影响

### 4.5 平台封禁时的分润

- 封禁触发的强平所产生的分润**冻结**，待调查结果决定是否发放

## 五、限制与风控

### 5.1 带单员风控告警

以下情况各触发 1 次风控告警：

1. 带单员同时**双向开仓**超过 **5 笔**
2. 带单员**带单仓位亏损累计时间**超过 **24 小时**未平仓
3. 带单员存在**多笔或多账号开仓并平仓时间趋于相同**（疑似操控）
4. 带单员近 **7 日无新增带单交易**

### 5.2 跟单交易限制

1. **体验金**不可用于抵消跟单交易的保证金与交易费用
2. 跟单者因**本金不足**跟单失败 **1 次**，系统**立即自动取消**跟随该带单员

### 5.3 跟单者数量

带单员的跟随者**不设数量上限**。

当某次跟单中，**带单金额总计（带单员 + 所有跟单者）超过平台该币种可开上限**时：
- 带单员优先占用额度
- 剩余额度按各跟单者金额占比 pro-rata 分配
- 分配后若某跟单者的实际可开金额 < 最小开仓量，该笔跟单跳过

## 六、开平仓规则

1. 带单员通过**专用带单路由**在交易页开仓，**限价、市价、计划单均支持**
2. 跟单者根据带单员**开仓成交订单**自动跟单；开仓订单成交多笔时**只跟单一次**
3. 带单员平仓后，当前仓位关联的所有跟单者仓位**全部平仓**；支持市价平仓、市价全平、止盈止损触发平仓
4. 带单员带单仓位被**强平**时，所有关联跟单者仓位同步**触及平仓**
5. 跟单者**主动停止跟单**后，所有跟单仓位**立即市价全平**，剩余资金转回

## 七、异常与边界

| 场景 | 处理 |
|------|------|
| 申请时仍有仓位/委托/计划单 | 申请失败 |
| 申请时仍有跟随者 | 申请失败 |
| 人工审核不通过 | 申请失败 |
| 跟单者资产不足最低跟单金额 | 无法跟随该带单员 |
| 带单员近 30 日无开仓记录 | 无法发起新跟随；已有跟随者系统自动取消跟随 |
| 跟单者单次本金不足 | 本次跟单失败，系统**立即**自动取消跟随 |
| 带单金额总计超平台该币种上限 | 带单员优先，剩余额度按跟单者占比 pro-rata 分配；分配后低于最小开仓量则跳过 |
| 带单员被移除跟随者 | 被移除者不再跟随新订单；已有仓位继续跟随带单员平仓；该部分仓位不计入分润 |
| 邀请码被 disable | 该码不可再被新跟随者使用；已通过该码加入的跟随者不受影响 |
| 跟单者主动停止跟单 | 所有跟单仓位立即市价全平，资金转回 |
| 带单员停止带单/注销 | 所有跟单者仓位立即市价全平；分润正常结算 |
| 平台封禁带单员 | 立即强平所有跟单仓位；冻结待结算分润；站内信通知跟单者 |
| 带单员带单仓位强平 | 所有关联跟单仓位同步平仓 |
| 账户级止损触发 | 所有跟单仓位市价全平，停止跟单，资金转回 |
| 锁定期内请求停止跟单或资金操作 | 操作被阻止；锁定期结束前不可执行 |
| 跟单进行中请求转出资金 | 操作被阻止；需先停止跟单（仓位全平）后方可转出 |
| 开仓订单成交多笔 | 跟单者只跟单一次（按首次成交触发） |
| 当日净盈利 ≤ 0 | 分润为 0，不触发预扣 |
