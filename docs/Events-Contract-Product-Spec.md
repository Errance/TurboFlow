# Events Contract（Now）产品规格文档

> **文档版本**：v1.0  
> **模块定位**：独立功能板块，与现有 Instant Market / Prediction Market 完全隔离  
> **当前阶段**：前端产品外形 + Mock 数据演示（不涉及后端 / 清结算 / 真实数据）

---

## 一、产品定位与体验目标

### 1.1 产品定位

Events Contract（Now）是一个短周期事件合约模块。用户对加密资产的短期价格走势进行预测（涨/跌），在极短的回合周期内（1 分钟 / 5 分钟 / 15 分钟）完成下注 → 持仓 → 结算的完整流程。

本模块的核心体验不同于传统交易终端——它更接近**回合制游戏 / 赌场**的沉浸式体验，强调节奏感、紧迫感和揭晓仪式感。

### 1.2 体验设计四原则

| 原则 | 描述 |
|------|------|
| **强节奏** | 每笔下注都是一个独立回合，从下注到结算的全过程有清晰的时间节奏 |
| **强反馈** | 所有关键时刻（下注成功、倒计时临近、结算揭晓、连胜）都有醒目的视觉反馈 |
| **强沉浸** | 结算揭晓默认打断当前操作，以全屏动画呈现结果（可由用户降级/关闭） |
| **低决策成本** | 用户能在 3–5 秒内完成「选时长 → 输金额 → 押方向」的决策链路 |

### 1.3 风格方向

- 主基调：**Casino Chip（赌场筹码）+ Cyber Minimal（赛博极简）**
- Casino Chip 贡献：回合仪式感、筹码/票券的重量感、结算揭晓的戏剧性
- Cyber Minimal 贡献：信息层级克制、排版秩序、避免视觉廉价感
- 所有视觉样式遵循现有 Design Spec（颜色、字体、圆角等），本文档仅定义结构与体验机制

## 二、产品范围

### 2.1 包含（Scope）

- 仅做 **Now** 模式
- 时间档位：**1min / 5min / 15min**
- 方向：**Higher / Lower**（不做 Range）
- 支持资产：**BTC/USDT, ETH/USDT**（下拉切换）
- 支持多笔订单并行（每单独立倒计时）
- 社交功能：全局玩家动态、排行榜（三榜）、微走势全局结算浮标
- 个人数据可视化：珠盘路（Bead Road）胜负图
- 动效可配置（用户可关闭/降级）

### 2.2 不包含（非目标）

- 不做 Weekly / Expiry 模式
- 不做 Big Win 大额盈利仪式（后续用排行榜/成就系统替代）
- 不做音效
- 不做后端真实清结算逻辑
- 不做动态赔率（本版本为基准赔率表 + 微调）

## 四、页面整体布局

### 4.1 三段式信息架构

页面由三个核心功能区域组成：

```
┌─────────────────────────────────────┐
│           Leaderboard（可折叠）       │
├──────────────────┬──────────────────┤
│    行情区         │    下单区         │
│  (Market Stage)  │  (Order Panel)   │
├──────────────────┤                  │
│    持仓区         │                  │
│ (Positions Stage)│                  │
└──────────────────┴──────────────────┘
```

### 4.2 桌面端布局

采用**响应式双列结构**：

- **顶部**：Leaderboard 可折叠区域（全宽）
- **次顶部**：价格条（左）+ Effects 设置入口（右），单行排列
- **主内容区**：双列网格
  - **左列**（弹性宽度）：微走势图 + 持仓区
  - **右列**（固定 380px）：下单面板，**sticky 定位**随页面滚动固定在视口内
- **微走势图与下单面板顶部对齐**

### 4.3 移动端布局

采用**单列纵向堆叠**：

```
Leaderboard → 行情区 → 下单面板 → 持仓区
```

- 微走势区域高度适当增大（相比桌面端更高的占比）
- 下单面板间距适当紧凑
- 持仓区默认可见 2–3 张卡片，可滚动查看更多

## 五、行情区（Market Stage）

行情区摒弃了传统的大 K 线图，改用高密度小组件组合。原因：对于 1/5/15 分钟的回合制下注，大 K 线信息冗余且降低刺激感密度。

### 5.1 价格条

展示当前选中资产的核心行情信息：

- **资产选择器**：下拉菜单，支持 BTC/USDT 和 ETH/USDT 切换。切换资产**不过滤现有持仓**
- **当前价格**：大号字体实时展示。价格变化时触发**微脉冲反馈**——数字短暂闪烁（涨绿、跌红），持续约 300ms
- **动量标签**：根据最近价格变化判断趋势方向（Bullish / Bearish / Neutral），以彩色文字标签形式展示。这是装饰性指示器，不追求交易级精度

### 5.2 微走势（Sparkline）

用精简的折线图替代 K 线，**无坐标轴、无网格**，只呈现价格走势轮廓：

- 折线颜色跟随涨跌状态（涨绿 / 跌红）
- 折线下方有淡色渐变填充，增强视觉层次
- 时间窗口与当前选中的 Time Increment 对齐（选 1min 显示最近 1 分钟走势，选 5min 显示最近 5 分钟，以此类推）
- 每 500ms 刷新一次

**微走势图上的叠加信息**（详见 §5.3 和 §5.4）：

### 5.3 用户下注标记

用户每次下单后，其入场点位会在微走势上标记：

**进行中的单子：**
- 在对应入场价格位置绘制**实心圆点**
- 圆点旁附带**方向三角**（Higher 向上 / Lower 向下）
- 标注入场价格文字标签

**已结算的单子：**
- 圆点变为**空心描边圆环**（视觉区分于进行中）
- 标签从入场价变为**结算成绩**（如 `W +$72` 或 `L -$20`）
- 标签背景为半透明绿色/红色
- 从入场点到结算价格之间绘制一条**水平虚线**连接，直观展示价格移动方向

**作用**：让用户直观看到自己在图上的"战场位置"，增强参与感和代入感。

### 5.4 全局结算浮标

微走势图上实时展示其他玩家的结算动态。当任意全局玩家的单子结算时，在微走势图右侧区域飘出一个浮动标签：

- **内容**：`{玩家名} +$XX` 或 `{玩家名} -$XX`
- **样式**：半透明圆角 pill 标签，盈利绿色底/亏损红色底
- **动态行为**——**散射效果**：
  - 盈利标签从当前价格线位置**向上飘动**
  - 亏损标签从当前价格线位置**向下飘动**
  - 同时伴随水平方向的微偏移，形成自然的散射感
  - 每个标签的偏移量基于其 ID 哈希计算，确保随机但稳定
- **动画周期**：4.5 秒，前半段保持高可见度，后半段逐渐淡出消失
- **同屏上限**：最多 5–6 个浮标同时展示，避免视觉混乱

**作用**：营造"热闹的交易大厅"氛围，让用户感知到全局交易活跃度，增强社交临场感。灵感来源于 Crash 类游戏的实时结算展示。

## 六、下单区（Order Panel）

下单面板的设计目标是让用户在 3–5 秒内完成完整的下注决策流程。

### 6.1 信息结构（从上到下）

1. **可用余额**：清晰展示在面板顶部右侧
2. **Time Increment 选择器**：1min / 5min / 15min 三档分段按钮，选中态有高亮
3. **金额输入区**：
   - 数字输入框 + 右侧 MAX 快捷按钮
   - 输入框上方明确展示金额范围限制：**$5 – $200 USDT**
   - 下方排列快捷筹码按钮：**$5 / $10 / $20 / $50 / $100**，点击直接填入对应金额
4. **收益预览**：左右两列对称展示 Higher / Lower 方向的预期收益
   - Higher 侧：绿色（teal）文字展示收益金额和回报率
   - Lower 侧：红色（pink）文字展示收益金额和回报率
   - **颜色严格与方向对应**——这是对标竞品的明确改进点
5. **风险提示**：弱化展示 `Loss if wrong: -$X.XX`
6. **双 CTA 按钮**：**Higher（绿）/ Lower（红）**，大按钮，占据面板底部全宽

### 6.2 下单交互

- Higher/Lower 按钮即**确认下注**，一键完成，**不提供 Undo**——强化"筹码上桌"的确定性
- 点击后按钮有 **active:scale 微缩**反馈
- 下单成功后弹出轻量 **Toast 提示**：`✓ BTC Higher · 100 USDT · 5min`，Toast 背景色对应方向（Higher 绿 / Lower 红），2 秒自动消失
- 下单后金额输入框自动清空，准备下一单
- **校验规则**：金额范围 $5–$200，余额不足时按钮禁用

### 6.3 赔率计算逻辑

- 基于固定 Strike 偏移表：1min ±0.15%，5min ±0.4%，15min ±0.8%
- 基准赔率表：1min 65%，5min 72%，15min 78%
- 方向差异化：赔率随价格与 Strike 的距离做微小浮动调整
- 用户输入金额后，两侧的收益预览**实时计算更新**

## 七、持仓区（Positions Stage）

### 7.1 整体结构

持仓区顶部为 Tab 切换栏，包含两个 Tab：

- **My Positions**：展示当前用户的活跃持仓卡片
- **All**：展示全局所有玩家的实时交易动态

**History 按钮**始终显示在 Tab 栏同行右侧（不随 Tab 切换隐藏），点击打开历史记录抽屉。

### 7.2 My Positions Tab

- 以**卡片列表**形式展示用户所有活跃持仓
- 默认按剩余时间升序排列（最快到期的在最前）
- 进入最后 5 秒的单子会**冒泡置顶**到列表最顶端（平滑过渡，不跳位）
- 空状态显示引导文案
- 移动端默认露出 2–3 张卡片，可滚动

### 7.3 All Tab（全局玩家动态）

实时展示所有玩家（mock 数据）的交易记录流：

- 每条记录：玩家头像（首字母）、玩家名、资产、方向标签、金额/PnL
- 记录状态：Active（灰色脉冲圆点）/ Won（绿色）/ Lost（红色）
- 新记录以**滑入动画**出现（从上方滑入）
- 最多展示最近 50 条记录
- 空状态：`Waiting for players...`

**作用**：让用户感受到平台上有大量真实玩家在同时参与，增强社交氛围和从众效应。

## 八、持仓卡片（Position Card）

每张持仓卡片是一个独立的"迷你回合舞台"，承载倒计时、PnL 实时变化和临战态动效。

### 8.1 卡片信息结构

- **顶部进度条**：全宽水平条，表示剩余时间占总时长的比例
- **方向标签**：HIGHER（绿底）/ LOWER（红底）
- **预估 PnL（核心大字）**：实时计算"如果此刻结算"的盈亏金额
  - 赢：绿色正数
  - 输：红色负数
  - 旁标 **EST.** + 小字 **"If settled now"** 降低误导
- **辅助信息四列**：Entry（入场价）/ Current（当前价）/ Amount（下注金额）/ Return（回报率）

### 8.2 PnL 实时翻转

- Estimated PnL 采用"二值翻转"逻辑：当前价在 Strike 之上/之下直接决定赢或输
- 赢 = +profit（金额 × 赔率）/ 输 = -amount
- 数字翻转时有微小缩放动效

### 8.3 倒计时动效时间轴

这是本产品的**核心体验差异点**——借鉴游戏中抽卡、开箱、孵蛋的递进式紧张感：

| 阶段 | 时间范围 | 表现 |
|------|---------|------|
| **常规态** | T > 5s | 进度条匀速消耗，颜色与方向一致（Higher 绿 / Lower 红），卡片静态 |
| **临战态** | 5s ≥ T > 0s | 进入高度紧张状态（详见下方） |
| **结算** | T = 0 | 卡片状态变为 settling，推入结算揭晓队列 |

### 8.4 临战态详细描述（最后 5 秒）

当剩余时间 ≤ 5 秒时，整张卡片进入临战态，所有效果同步触发：

1. **进度条**：颜色变为红色，开始脉冲发光（亮度在亮红和暗红之间交替闪烁）
2. **大号倒计时数字**：在卡片右上角出现红色的秒数（5、4、3、2、1），每个数字出现时有**弹出缩放动效**
3. **卡片整体震动**：轻微左右抖动，模拟"即将开奖"的紧张感
4. **红色边框**：卡片边框变为实心红色
5. **呼吸发光阴影**：卡片外围有红色的渐变发光，缓慢脉冲
6. **红色渐变背景**：卡片背景出现微妙的红色渐变

**不使用任何徽章/贴片元素**——紧迫感完全通过时间容器（进度条 + 数字）和卡片整体效果来表达，这是比徽章更高级、更克制的方式。

## 九、结算揭晓（Settlement Reveal）

### 9.1 设计理念

结算揭晓是**打断式沉浸体验**——当任何一笔订单结算时，默认弹出全屏揭晓动画，不论用户当前在做什么。这不是 bug，这是回合制体验的核心设计。但必须提供关闭选项兜底。

### 9.2 全屏揭晓动画序列

通过全屏遮罩覆盖整个视口，背景半透明暗色模糊，保留底层页面若隐若现：

| 时间点 | 阶段 | 内容 |
|--------|------|------|
| 0ms | 遮罩入场 | 暗色遮罩 + 背景模糊淡入 |
| 80ms | 结果标题 | **WIN** 或 **LOSS** 大字入场 |
| 450ms | PnL 数字爆出 | 盈亏金额大号数字弹跳出现 |
| 900ms | 交易摘要 | 三行辅助信息淡入 |
| 1600ms | 持续展示 | 全部内容静止展示 |
| ~6800ms | 自动退出 | 淡出消失（或用户点击关闭） |

### 9.3 揭晓信息结构

```
          WIN                    ← 结果标题（大号、发光）

        +$72.00                  ← PnL 数字（超大号、弹跳入场）

    BTC/USDT · Higher            ← 资产 + 方向
  Entry $87,432 → Settled $87,650  ← 入场价 → 结算价
       Bet $100 · 5min           ← 下注金额 + 时长
```

### 9.4 胜负视觉差异

**WIN 状态（胜利）：**
- 背景径向渐变：从中心向外的绿色（teal）脉冲扩散
- PnL 数字周围散出 **8–12 个小光点粒子**，从中心向外扩散并淡出
- PnL 数字下方有一条**横向光线扫过**效果
- 标题从底部弹入并微旋转回正，给出"盖章"的力量感

**LOSS 状态（失败）：**
- 背景径向渐变：从中心向外的暗红色，**更克制**（输不要太刺眼/打击）
- **无粒子效果**
- 标题简单淡入下滑

### 9.5 退出与队列

- 用户**点击屏幕任意位置** 或按 **ESC** 可立即关闭当前揭晓
- 关闭后如果队列中还有其他已结算的单子，**立即无缝衔接**展示下一个
- **跳过只跳一个**——用户必须逐个关闭，不能一次跳过全部
- 多单同时到期时，按时间顺序排队，**绝不合并**
- 动画结束后约 5 秒自动关闭（如用户未操作）

### 9.6 效果等级

| 等级 | 名称 | 行为 |
|------|------|------|
| **On** | 全屏揭晓 | 完整的全屏遮罩 + 动画序列（默认） |
| **Minimal** | 卡片级揭晓 | 不全屏，仅在持仓卡片内部播放简短闪烁效果，约 800ms |
| **Off** | 静默更新 | 纯静态结果更新，卡片直接消失进 History |

## 十、仪式感系统（Ceremony）

### 10.1 触发条件

仪式层是叠加在普通结算揭晓之上的额外强化动效：

| 触发事件 | 条件 | 仪式类型 |
|---------|------|---------|
| **连胜 Streak** | 连续赢 ≥ 3 次 | 连胜仪式 |
| **逆转 Comeback** | 连续输 ≥ 3 次后赢 1 次 | 逆转仪式（触发后连胜计数重置为 1） |

### 10.2 仪式表现

在结算揭晓动画播放完毕后，额外叠加：

- **射线效果**：16 条从中心向外放射的光线（连胜用绿色，逆转用金色）
- **外环脉冲**：200px 圆环从中心扩散，重复 2 次
- **徽章/盖章**：
  - 连胜：`WIN STREAK · {N}x`
  - 逆转：`COMEBACK · CLUTCH`
  - 徽章有弹跳入场动画 + 轻微上下浮动
- **持续时间**：约 3 秒（2 次循环），可点击跳过
- 受 **Hype Mode** 开关控制

### 10.3 关键原则

- 仪式层**不引入任何持久性徽章**——仪式仅在揭晓瞬间展示，不会驻留在卡片或其他 UI 上
- 仪式的目的是给用户"高光时刻"的记忆锚点，而非持续的视觉噪音

## 十一、Effects 设置

### 11.1 可发现性

- 在 Events 页面右上角提供 **"Effects"** 按钮入口
- 首次结算后弹出一次性提示：可在 Effects 中关闭/降级动效

### 11.2 设置项

| 设置 | 选项 | 默认值 | 说明 |
|------|------|--------|------|
| **Settlement Reveal** | On / Minimal / Off | On | 控制结算揭晓的强度等级 |
| **Hype Mode** | On / Off | On | 控制连胜/逆转仪式层是否启用 |

### 11.3 设计理由

动效默认开启最强档位，因为 Events Contract 的核心体验就是刺激感。但**必须提供关闭选项**：
- 频繁使用的用户会产生疲劳
- 部分用户天然不喜欢被打断
- 这是产品尊重用户偏好的必要选项

## 十二、排行榜（Leaderboard）

### 12.1 展示形式

排行榜以**页面内可折叠区域**的形式嵌入 Events 页面最顶部：

- **收起态**：一行紧凑按钮，展示奖杯图标 + "Leaderboard" + 当前 #1 玩家简要信息
- **展开态**：全宽展开，将下方所有内容推下。展示完整的排行榜表格

### 12.2 三个独立榜单

排行榜包含三个独立的 Tab，每个 Tab 是一份独立排名：

| Tab | 排序字段 | 主指标 | 辅助指标 |
|-----|---------|--------|---------|
| **Total Profit** | 总盈利金额 | Total Profit（加粗绿色） | Streak, Best Trade |
| **Best Trade** | 单笔最高盈利 | Best Trade（加粗绿色） | Streak, Total Profit |
| **Win Streak** | 最高连胜记录 | Win Streak（加粗绿色） | Best Trade, Total Profit |

### 12.3 交互

- 切换 Tab 时重置到第 1 页
- 每页展示 10 条记录
- 底部分页控件：`< 1 2 3 4 5 >`
- 前 3 名有金/银/铜视觉高亮
- 共约 50 名玩家的排行数据

## 十三、历史记录与珠盘路

### 13.1 History Drawer

点击 "History" 按钮打开历史记录面板：

- 桌面端：居中弹出圆角面板
- 移动端：底部滑出面板
- 最大高度：视口高度的 80%

内容结构（从上到下）：

1. **汇总统计**：Total P&L / Win Rate / W–L 数量
2. **珠盘路（Bead Road）**
3. **逐笔记录列表**

### 13.2 珠盘路（Bead Road Chart）

借鉴百家乐珠盘路的标准网格可视化，用于直观展示用户的胜负走势：

- **网格结构**：6 行 × N 列，标准棋盘式网格
- **填充顺序**：从上到下、从左到右（第 1–6 笔填入第 1 列，第 7–12 笔填入第 2 列，以此类推）
- **每格**：24×24px 正方形单元格，带细边框
- **圆点**：16px 纯色实心圆，居中于单元格
  - 绿色 = 赢
  - 红色 = 输
  - **无 W/L 文字**（标准珠盘路只用颜色区分）
- **空格**：如果最后一列未满，空位显示为带边框的空单元格
- **横向滚动**：当列数超出容器宽度时可水平滚动，自动滚动到最右侧（最新数据）
- **标题**：`BEAD ROAD` + 总场次数

**作用**：让用户一眼识别出自己的胜负连续性趋势，比纯列表更直觉化。这是赌场类产品的标准配置元素。

## 十四、全局玩家数据系统

### 14.1 Mock 玩家引擎

系统在后台持续生成模拟的全局玩家活动：

- 预设约 60+ 个加密风格假用户名
- 每 2–4 秒随机生成一笔假下单记录
- 每笔假单在 8–15 秒后自动结算
- 排行榜数据每 30 秒微调一次，制造"活"感

### 14.2 数据消费方

| 数据 | 消费者 | 用途 |
|------|--------|------|
| 全局下单流 | All Tab (GlobalFeed) | 展示实时交易动态 |
| 全局结算事件 | 微走势浮标 (SettlementFloat) | 展示结算成绩飘字 |
| 排行榜数据 | Leaderboard | 展示三榜排名 |

### 14.3 目的

所有全局数据的目的是**营造热闹的交易氛围**：
- 让用户感知到"有很多人在玩"
- 制造 FOMO（害怕错过）效应
- 通过排行榜提供追赶目标感

## 十五、赔率与结算规则

### 15.1 赔率体系

| 时长 | Strike 偏移 | 基准赔率 |
|------|------------|---------|
| 1 min | ±0.15% | 65% |
| 5 min | ±0.40% | 72% |
| 15 min | ±0.80% | 78% |

- 用户选择 Higher 方向时，Strike = 当前价格 × (1 + 偏移百分比)
- 用户选择 Lower 方向时，Strike = 当前价格 × (1 - 偏移百分比)
- 实际赔率在基准赔率基础上随价格与 Strike 的距离做 ±2% 的微调

### 15.2 结算规则

- **Higher 赢**：结算价 > Strike Price
- **Lower 赢**：结算价 < Strike Price
- **盈利** = 下注金额 × 赔率
- **亏损** = 下注金额（全额损失）

### 15.3 金额限制

- 单笔下注范围：**$5 – $200 USDT**
- 初始模拟余额：**$10,000 USDT**

## 十六、交互边界与异常处理

（因为不做后端，本节仅定义 UI 外形应如何表现）

| 场景 | 表现 |
|------|------|
| 数据延迟 / 价格不更新 | Estimated PnL 显示 `—` 或 `Updating…` |
| 倒计时与结算之间 | 可显示 `Settling…` 状态（短暂） |
| 多单同时到期 | 按时间顺序逐个弹出揭晓，不合并 |
| 余额不足 | Higher/Lower 按钮禁用，金额输入无法超出余额 |
| 金额超出范围 | 不允许输入低于 $5 或高于 $200 |
